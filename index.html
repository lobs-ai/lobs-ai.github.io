The Origin

One dev.
An entire team of agents.

What started as a simple OpenClaw assistant has evolved into a full multi-agent orchestration platform. Built by Rafe — a UMich CSE grad student who juggles classes, GSI duties, and a summer Microsoft internship — all while operating like a one-person startup.

The goal was never just to have an AI assistant. It was to have a workforce — specialized agents that run autonomously, pick up tasks, and deliver results while he's in lecture, at office hours, or asleep.

v1

Simple OpenClaw Assistant

Single agent, manual prompts, basic tool use. The foundation — but the ceiling was obvious fast.

v2

lobs-server + Python Orchestrator

FastAPI backend with a Python-based multi-agent orchestrator. Powerful, but 38K lines of Python that needed constant tending.

v3

PAW Plugin — TypeScript Rewrite

Replaced the Python orchestrator with a TypeScript OpenClaw plugin. 79% code reduction — 38K lines → 8K lines. Workflow engine, model chooser, circuit breaker, all built-in.

Build Log

Wins & Hard-Won Lessons

Real systems break in real ways. Here's what worked and what didn't.

🏆 Wins & Milestones

⚙️

Workflow Engine as Control Plane

Multi-step tasks now run as structured workflows with state, branching, and rollback — not just one-shot prompts.

📉

79% Code Reduction

Python → TypeScript rewrite. 38K lines became 8K. The system became more capable, not less.

🎯

Model Tier System

Tasks automatically route to the cheapest model that can handle them, with fallback chains on failure.

🪞

Agent Reflection & Learning

Workers reflect on completed tasks, capture lessons, and feed back into the shared memory system.

🔌

Circuit Breaker Patterns

Automatic failure detection prevents cascading issues. Workers that fail too often get quarantined and escalated.

💀 Losses & Lessons

🔄

The Restart Loop Incident

Workers edited plugin source code, then called gateway restart. The restart spawned new workers. Who edited source. Infinite loop. Gateway restart is now denied for all workers.

💬

Heartbeat Spam

Background exec commands in workers triggered event-driven heartbeats on completion — flooding the main session with noise. Banned all background exec patterns.

🐛

JSON Double-Conversion Bug

Swift's .convertFromSnakeCase + manual CodingKeys caused silent field drops. Lesson: never add CodingKeys for simple snake→camel conversions.

👻

Orphaned Session Errors

"No tool call found" errors from workers that made no tool calls before completing. Required stricter session validation.

🔁

Reflection Spam

Agent reflection was too eager — workers would reflect after every minor action. Now gated behind strict completion criteria and rate limits.
